# 产品需求文档（PRD）

> 定义 MedAI Learn 的产品目标、用户故事与功能清单。

---

## 1. 产品概述

**产品名称**：MedAI Learn（医疗 AI 助手）

**核心目标**：
1. 作为学习载体，覆盖 AI 应用工程师所需的全栈技术
2. 提供实用的医疗 AI 功能，具备真实业务价值

**目标用户**：
- 用药查询人群（查询药品说明书、相互作用）
- 慢病管理人群（用药提醒、健康记录）
- 医疗从业者（辅助诊断参考、文献检索）

## 2. 用户故事

### Epic 1：用户认证

| ID | 角色 | 故事 | 验收标准 |
|----|------|------|---------|
| US-101 | 新用户 | 我想通过邮箱注册账号 | 邮箱验证 + 密码强度检查 |
| US-102 | 新用户 | 我想通过微信/Google 一键登录 | OAuth 2.0 流程 + 首次登录绑定 |
| US-103 | 已登录用户 | 我想保持登录状态 | JWT + Refresh Token 自动续期 |
| US-104 | 已登录用户 | 我想安全退出 | Token 失效 + 清除本地状态 |

### Epic 2：药品查询

| ID | 角色 | 故事 | 验收标准 |
|----|------|------|---------|

| US-202 | 用户 | 我想查看药品说明书 | 结构化展示（成分/用法/禁忌） |

### Epic 3：AI 对话

| ID | 角色 | 故事 | 验收标准 |
|----|------|------|---------|

| US-302 | 用户 | 我想看到 AI 实时回复 | Streaming 流式输出 |
| US-303 | 用户 | 我想查看回答的依据来源 | RAG 引用标注 + 原文链接 |
| US-304 | 用户 | 我想保存对话历史 | 会话管理 + 历史搜索 |

### Epic 4：用药提醒

| ID | 角色 | 故事 | 验收标准 |
|----|------|------|---------|

| US-402 | 用户 | 我想收到提醒通知 | 浏览器通知 + 邮件（可选） |
| US-403 | 用户 | 我想记录服药状态 | 已服/跳过/延迟 + 统计 |

### Epic 5：健康档案

| ID | 角色 | 故事 | 验收标准 |
|----|------|------|---------|

| US-502 | 用户 | 我想让 AI 考虑我的健康背景 | 对话时自动注入个人档案 |
| US-503 | 用户 | 我想导出我的健康数据 | JSON/PDF 导出 |

## 3. 功能优先级

### P0 — 必须有（Phase 1-2）

- [ ] 用户注册/登录（邮箱 + OAuth）
- [ ] 药品搜索与详情展示
- [ ] AI 对话（基于药品知识库的 RAG）
- [ ] Streaming 流式输出

### P1 — 应该有（Phase 3）

- [ ] 药品相互作用查询
- [ ] 对话历史管理
- [ ] 用药提醒系统
- [ ] 个人健康档案

### P2 — 可以有（Phase 4）

- [ ] 多语言支持
- [ ] 药品图片识别（OCR）
- [ ] 健康数据可视化
- [ ] 社区问答

## 5. 数据模型（概览）

```
User
├── id, email, name, avatar
├── healthProfile (1:1)
│   ├── allergies: string[]
│   ├── conditions: string[]
│   └── currentMedications: string[]
└── conversations (1:N)
    ├── id, title, createdAt
    └── messages (1:N)
        ├── role: user | assistant
        ├── content: string
        └── citations: DrugReference[]

Drug
├── id, name, genericName
├── description, usage, dosage
├── sideEffects, contraindications
└── interactions (N:N)

Reminder
├── userId, drugId
├── schedule: cron expression
└── status: active | paused | completed
```

## 6. 约束与假设

### 约束

- 药品数据来源：公开药品说明书（非实时更新）
- AI 回答必须标注「仅供参考，不构成医疗建议」
- 不支持在线购药功能
- 不支持医患实时通讯

### 假设

- 用户主要使用中文交互
- 用户有基本的互联网使用能力
- 药品数据库规模：初期 ~1000 种常用药
