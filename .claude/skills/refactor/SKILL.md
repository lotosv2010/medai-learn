# /refactor — 学习式重构

用法：`/refactor` 重构当前文件 | `/refactor <目标>` 按目标重构

## 执行步骤

**重构前：先分析，再动手**

### Step 1：代码体检
分析当前代码的问题，分类输出：
- 📦 **结构问题**：职责不单一、层级混乱
- 🎯 **类型问题**：any 滥用、类型不准确
- ⚡ **性能问题**：不必要的渲染、大对象克隆
- 🔧 **可维护性**：命名不清晰、魔法数字、重复逻辑

### Step 2：重构方案
给出 2 种重构方案，标注各自的权衡：
- 方案 A：保守重构（最小改动，降低风险）
- 方案 B：理想重构（架构层面的改进，需要更多时间）

询问用户选择哪种，默认选 A。

### Step 3：逐步执行
每改一个部分，用注释说明：
```typescript
// 🔄 重构：将 any[] 改为 Drug[] 类型
// 原因：明确类型让 TypeScript 能在编译期捕获错误
// 知识点：TypeScript 的类型收窄（Narrowing）
```

### Step 4：重构后对比
展示 before / after 对比，量化改进：
- 类型安全性：❌ → ✅
- 代码行数：100 行 → 70 行
- 函数复杂度：高 → 低

### Step 5：学习提炼
此次重构涉及的设计模式或原则：
- SOLID 中的哪条原则？
- 这个模式叫什么名字？
- 在 medai-learn 的其他地方还能用到吗？

---

**注意**：重构完成后，建议用 `/learn <模式名>` 记录这次学到的重构技巧。
