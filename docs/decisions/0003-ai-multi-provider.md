# ADR-0003：AI 模型多供应商策略

- **状态**：已接受
- **日期**：2026-05-26
- **决策者**：项目组

---

## 背景

项目需要接入多个 AI 模型供应商（OpenAI、Anthropic、通义千问），需要决定如何抽象和管理多模型调用。

## 决策

采用 **Vercel AI SDK 统一接口 + 自定义路由层** 的方案。

## 架构

```
packages/ai-sdk/
├── providers/          # 各供应商适配器
│   ├── openai.ts
│   ├── anthropic.ts
│   └── dashscope.ts
├── router.ts           # 模型路由（根据场景选择模型）
├── types.ts            # 统一类型定义
└── index.ts            # 导出
```

## 理由

1. **学习价值**：Vercel AI SDK 是当前最流行的 AI 应用框架，掌握它对求职有直接帮助
2. **可替换性**：统一接口使得切换模型零成本
3. **成本控制**：不同场景使用不同模型，优化成本
4. **降级策略**：主模型不可用时自动切换备用模型

## 路由策略

| 场景 | 模型 | 理由 |
|------|------|------|
| 简单药品查询 | 通义千问 | 成本低、中文好 |
| 复杂用药分析 | GPT-4o | 推理能力强 |
| 敏感医疗问题 | Claude | 安全性高、拒绝率低 |
| Embedding | text-embedding-3-small | 性价比高 |

## 影响

- 需要学习 Vercel AI SDK 的 Provider 抽象
- 需要实现统一的错误处理和重试逻辑
- 需要监控各模型的 Token 用量和成本
