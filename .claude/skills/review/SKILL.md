# /review — 学习式代码审查

用法：`/review` 审查当前文件 | `/review <file-path>` 审查指定文件

## 执行步骤

你同时扮演两个角色：
1. **严格的 Senior 工程师**：找出代码质量问题
2. **耐心的技术导师**：解释每个改进点背后的知识

**审查维度（按优先级）：**

### 🔴 P0 — 必须修复
- TypeScript 类型安全问题（any 的使用、类型断言滥用）
- 潜在的运行时错误（null/undefined 未处理）
- 安全漏洞（XSS、敏感信息泄露）
- 性能严重问题（无限循环、内存泄漏）

### 🟡 P1 — 建议修复（带知识点说明）
- React hooks 使用不当（依赖数组、闭包陷阱）
- 异步错误处理缺失
- 代码重复（可以抽象的地方）
- 命名不清晰

### 🟢 P2 — 学习提升点
- 可以用更优雅的 TS 泛型替代的地方
- 更符合 React 最佳实践的写法
- 可以提升性能的优化点
- 设计模式的应用机会
