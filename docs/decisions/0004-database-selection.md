# ADR-0004：数据库选型（PostgreSQL + pgvector）

- **状态**：已接受
- **日期**：2026-05-26
- **决策者**：项目组

---

## 背景

项目需要关系型数据存储（用户、药品、对话）和向量检索（RAG），需要决定数据库方案。

## 决策

采用 **PostgreSQL + pgvector 扩展** 的一体化方案。

## 备选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A. PostgreSQL + pgvector** | 一体化、SQL 生态成熟、事务支持 | 需要配置扩展 |
| B. PostgreSQL + Pinecone | 向量检索专业 | 多一个服务、数据同步复杂 |
| C. MongoDB Atlas | 文档灵活、内置向量 | 不适合关系型数据 |
| D. Supabase | 开箱即用 | 学习价值低（封装太深） |

## 理由

1. **学习价值**：PostgreSQL 是最流行的关系型数据库，pgvector 是向量检索的主流方案
2. **一体化**：关系型数据和向量数据在同一数据库中，简化架构
3. **SQL 能力**：复杂查询、事务、索引优化都是重要学习点
4. **生态成熟**：Drizzle ORM 提供类型安全的 SQL 操作

## 影响

- 需要学习 pgvector 的向量索引（IVFFlat / HNSW）
- 需要掌握 Drizzle ORM 的 Schema 定义和迁移
- 需要理解向量相似度搜索的原理
